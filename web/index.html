About AgentOS

AgentOS is a production-ready LLM API routing gateway with intelligent routing and privacy protection. Inspired by vLLM Semantic Router architecture, AgentOS provides a unified interface to route requests across 16+ LLM providers including OpenAI, Anthropic, AWS Bedrock, Azure OpenAI, Google Gemini, Vertex AI, and major Chinese LLMs (DeepSeek, Qwen, Baichuan, Zhipu, Moonshot, MiniMax), plus local inference engines (vLLM, Ollama, LM Studio).
At its core, AgentOS solves three fundamental challenges: provider fragmentation, cost optimization, and data protection. By providing a single OpenAI-compatible API endpoint with transparent proxy support, AgentOS enables developers to switch providers, optimize costs, and protect sensitive data without changing a single line of application code.

Key Features

  • Multi-Provider Routing: 16+ providers with automatic format conversion (OpenAI ↔ Anthropic Messages API)
  • Transparent Proxy Mode: mitmproxy-based interception for zero-config integration (Envoy-style architecture)
  • Smart Routing: Keyword-based and wildcard pattern matching with priority-based rule engine
  • Cost Optimization: Automatic model downgrade for simple tasks (e.g., gpt-4 → gpt-4o-mini for translations)
  • Hot Reload: Update configuration without restarting (file-watch automation)
  • Web Dashboard: Vue 3 + Tailwind UI with real-time logs, configuration editor, and test interface
  • Privacy Guard: Three-layer security with PII masking, regex audit, and policy enforcement
  • Streaming Support: Full SSE (Server-Sent Events) passthrough for all providers

Architecture Highlights

AgentOS implements an Envoy-style transparent proxy architecture using mitmproxy + FastAPI. The system intercepts HTTPS traffic, extracts routing decisions via a Python addon (similar to Envoy ExtProc filter), and forwards requests to the appropriate LLM provider. This design is inspired by vLLM Semantic Router but simplified for Python-based deployments.
Key components: Router Server (FastAPI on port 8001) handles OpenAI-compatible API, routing logic, and dashboard. Proxy Server (mitmproxy on port 8801) intercepts client traffic transparently. Unified LLM Client supports all 16+ providers with automatic type detection, format conversion, and authentication handling. DNS Resolver bypasses routing loops by resolving real IPs directly.

Provider Support

International: OpenAI, Anthropic, AWS Bedrock, Azure OpenAI, Google Gemini, Vertex AI. Chinese LLMs: DeepSeek, Qwen (Alibaba), Baichuan, Zhipu (GLM), Moonshot, MiniMax. Local Inference: vLLM, Ollama, LM Studio (all support OpenAI-compatible API with no authentication required).

Technical Stack

Backend: Python 3.9+, FastAPI (async ASGI), httpx (HTTP client), mitmproxy (transparent proxy), PyYAML (configuration). Frontend: Vue 3 (Composition API), Tailwind CSS, vanilla JavaScript (no build step required). Testing: pytest + comprehensive integration tests for all 16+ providers.
